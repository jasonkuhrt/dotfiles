Research on Zed's profile system for switching between settings configurations.

**Last verified**: 2026-02-15

## Overview

Settings profiles were introduced in [PR #35339](https://github.com/zed-industries/zed/pull/35339) (merged ~Aug 2025).

**Key behavior**: Profiles **extend/overlay** base settings - they don't replace entirely. When activated, profile settings override base config until toggled off.

## Configuration

Defined in `settings.json` under the `profiles` key:

```json
{
  "profiles": {
    "focus": {
      "buffer_font_size": 16,
      "theme": "Tokyo Night",
      "tab_bar": { "show": false },
      "toolbar": { "breadcrumbs": false, "quick_actions": false },
      "gutter": { "line_numbers": false }
    },
    "presentation": {
      "buffer_font_size": 24,
      "ui_font_size": 20,
      "theme": "One Light"
    }
  }
}
```

## Switching Profiles

| Method | How |
|--------|-----|
| Command palette | Type "switch profiles" or "settings profile selector" |
| Action name | `settings_profile_selector::Toggle` |
| Default keybind | `ctrl+opt+cmd+P` (macOS) |
| UI | Click account picture in top-right corner |

### Custom Keybinding

Add to `keymap.json`:

```json
{
  "bindings": {
    "ctrl-shift-p": "settings_profile_selector::Toggle"
  }
}
```

## What CAN Be Included in Profiles

- Any top-level Zed setting
- Font settings (`buffer_font_size`, `ui_font_size`, `buffer_font_family`)
- Theme (`theme`)
- UI visibility (`tab_bar`, `toolbar`, `gutter`, `scrollbar`)
- Editor behavior (`soft_wrap`, `preferred_line_length`)
- Nested settings objects

Full autocomplete works when editing profiles in Zed.

## What CANNOT Be Included

| Feature | Supported | Notes |
|---------|-----------|-------|
| Keybindings | NO | Keymaps are separate from settings system |
| Extensions | NO | Can't selectively load/unload per profile |
| Agent profiles | Separate | Use `agent.profiles` instead |
| Language settings | Untested | May work but not documented |

## Known Limitations

1. **No profile-specific keybindings** - Keymaps don't support profiles
2. **No default profile setting** - Always starts with base settings
3. **No CLI launch flag** - Can't do `zed --profile focus`
4. **Double-save bug** - Some nested settings may require saving twice
5. **No profile isolation** - API keys, history shared across profiles

## Agent Profiles (Different System)

Zed has a separate "agent profiles" system for AI tool configuration:

```json
{
  "agent": {
    "profiles": {
      "write": { /* tools config */ },
      "ask": { /* read-only tools */ }
    }
  }
}
```

These are unrelated to settings profiles.

## Example: Focus Profile

Since Zed lacks true zen mode ([see research](./2026-02-15-2320-zed-zen-mode-status.md)), a profile can approximate it:

```json
{
  "profiles": {
    "focus": {
      "tab_bar": { "show": false, "show_nav_history_buttons": false },
      "toolbar": { "breadcrumbs": false, "quick_actions": false },
      "gutter": { "line_numbers": false, "folds": false },
      "indent_guides": { "enabled": false },
      "current_line_highlight": "none",
      "scrollbar": { "show": "never" },
      "centered_layout": { "left_padding": 0.25, "right_padding": 0.25 },
      "buffer_font_size": 14
    }
  }
}
```

**Limitation**: Title bar and status bar still visible (see [#5120](https://github.com/zed-industries/zed/issues/5120) - OPEN).
