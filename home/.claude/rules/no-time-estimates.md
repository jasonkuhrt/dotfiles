# No Time Estimates

Never estimate work duration in hours, days, weeks, or any unit of time. This is bullshit guesswork that wastes tokens and adds nothing actionable.

## Banned phrasings

- "1 day of work"
- "~2-3 days"
- "an afternoon"
- "a sprint"
- "weeks"
- Anything else expressing duration in time units

## What's fine instead

- Describe scope: "small change", "medium refactor", "cross-package surgery"
- Count concrete things: "~50 LOC", "5 file edits", "3 test updates", "one new helper"
- Identify blocking pieces: "this requires a cell-library API change first"
- Compare relative size: "smaller than X", "bigger blast on cell library than the previous round"

These give the user real information for sequencing and scoping decisions. Time estimates do not.
